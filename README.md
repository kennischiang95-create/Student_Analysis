# 110～114學年度生源學校分析 Dashboard

## 開啟方式

直接雙擊根目錄的 `index.html` 即可開啟。圖表套件 Chart.js 透過 CDN 載入，因此需要網路連線。

## 專案結構

```text
student-dashboard/
├── index.html
├── css/
│   └── style.css
├── js/
│   ├── data.js      # 本網站使用的生源學校人數
│   ├── config.js    # 區域規則與三組已確認同校對照
│   └── app.js       # 圖表、篩選、表格與下載功能
├── .nojekyll
└── README.md
```

網站不會讀取或連結 Excel，只包含已放入 `js/data.js` 的本專案資料。

## 上傳 GitHub Pages

1. 在 GitHub 建立新的 Repository。
2. 將本資料夾內的所有檔案與子資料夾上傳到 Repository 根目錄；不要只上傳 `index.html`。
3. 到 Repository 的 **Settings → Pages**。
4. Source 選擇 **Deploy from a branch**。
5. Branch 選擇 `main`，資料夾選擇 `/ (root)`，再按 Save。
6. GitHub 產生的 Pages 網址即可分享給其他人。

## 已套用的資料規則

- 原始校名與人數保留。
- 只有三組使用者已確認為同校的名稱會在「合併已確認同校」模式合併。
- 原始空白校名與「高級中等教育階段非學校型態實驗教育」保留人數，但不納入生源學校數及縣市辨識。
- 完整資料表不顯示110→114差值。
- 成長與衰退統一使用「110→114」的閱讀方向。

## 本次功能更新

- 資料品質摘要保留原始空白校名與非學校型態資料，不再顯示沒有內容的「其他未辨識校名」項目。
- 年度切換改採局部更新與圖表資料更新，不再重建所有頁籤、完整資料表及全部圖表，以降低操作卡頓。
- 總覽折線圖會依區域、縣市、搜尋學校與學年度條件即時更新。
- 「學年度／總計」改為複選器：可選五年總計，或同時選擇多個學年度，例如111、112、113。
- 選擇特定年度後，排行榜、縣市／區域圖、KPI與單校趨勢都會依所選年度重新計算。
- 五年總計模式下，趨勢圖仍顯示110～114五個年度的變化；多年度模式只顯示勾選的年度。
